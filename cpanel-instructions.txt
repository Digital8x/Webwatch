# WebWatch Pro - cPanel Git Deployment Instructions

Since you have Git and Terminal access on your BigRock cPanel, deploying is much cleaner and faster! Follow these exact steps.

## Step 1: Clone the Repository
1. Log into your BigRock cPanel and open the **Terminal**.
2. Run this command to download your code directly from GitHub into a new folder:
   `git clone https://github.com/Digital8x/Webwatch.git webwatch-server`
3. Enter the folder:
   `cd webwatch-server`

## Step 2: Add Your Password (Crucial!)
Because we specifically blocked your password from being uploaded to GitHub for security, you must recreate it on the server.
1. Run this command in the terminal to create your secret `.env` file:
   `echo "WEBWATCH_PASSWORD=W@tch!Pr0#9kL2$mX8" > .env.local`

## Step 3: Install & Build
Still inside the terminal, run:
1. `npm install`
2. `npm run build`
*(This installs the dependencies and compiles your Next.js application for production).*

## Step 4: Start the Server
Depending on how you manage Node apps on BigRock:
**Option A (Using PM2 - Recommended):**
`pm2 start npm --name "webwatch" -- start`
`pm2 save`

**Option B (Using cPanel's Setup Node.js App UI):**
1. Go to **Setup Node.js App** in cPanel.
2. Create an App pointing to the `webwatch-server` folder.
3. Set the startup file to `node_modules/next/dist/bin/next`.
4. Click Start.

## Step 5: The 24/7 Cron Job (CRITICAL)
For the server to actually check your websites 24/7, you must set up a cron job.
1. Go back to the main cPanel dashboard.
2. Click on **Cron Jobs** (under Advanced).
3. Under "Common Settings", select **Once Per Minute (* * * * *)**.
4. In the **Command** field, paste this exactly (replace with your actual URL):
   `curl -X POST https://monitor.yourdomain.com/api/checks/run >/dev/null 2>&1`
5. Click **Add New Cron Job**.
